# 技术点学习文档

这个目录用于记录 `apps/page` 中采用的技术方案、项目内具体用法，以及和其它常见方案的优缺点比较。

`page` 的定位不是堆功能，而是把后台管理前端的关键技术点拆开学习。因此这里的文档优先回答三个问题：

1. 这个技术点在当前项目里怎么用？
2. 为什么采用这个方案？
3. 和其它方案相比有什么取舍？

## 当前学习主线

```text
Axios
  -> 负责 HTTP 请求、双 Token 刷新、错误归一化

Pinia Colada
  -> 负责 query / mutation 的状态、缓存、刷新、失效

Pinia
  -> 负责本地登录态、用户快照、权限、布局等状态

Vite+
  -> 负责开发、构建、检查、修复、测试命令入口
```

## 记录原则

| 原则     | 说明                                                 |
| -------- | ---------------------------------------------------- |
| 对照源码 | 每篇文档必须指出当前项目中对应的源码位置             |
| 比较方案 | 不只写“怎么用”，也写“为什么不用另一种方案”           |
| 保持边界 | 区分当前已实现内容和后续预留内容                     |
| 面向复习 | 文字以理解技术方案为主，不做过度封装，不照抄框架文档 |
| 持续更新 | 后续实现用户管理、角色权限、菜单权限时继续补充子文档 |
