# 后台管理前端学习地图

## 1. 这份地图解决什么问题

学习一个后台管理前端时，很容易按照页面或技术名词罗列内容：

```text
登录、表格、弹窗、Vue、Pinia、Axios、RBAC……
```

这种方式混合了页面形态、框架、工具和业务能力，不容易形成稳定的知识结构。

本项目改用五个前端方向组织学习：

```text
视图呈现、应用组织、数据交互、用户与权限、工程与交付
```

它们描述的是前端需要承担的五类职责，不与某个具体框架绑定。Vue、React、Solid
只是“应用组织”的不同实现方式；表格、表单、弹窗等则是视图呈现深入学习时接触的
具体 UI 组件形态。

## 3. 详细学习树

### 3.1 视图呈现

```text
视图呈现
├─ 布局
│  ├─ 应用布局
│  │  ├─ 侧边栏、顶部栏、内容区
│  │  ├─ 固定区域与滚动区域
│  │  └─ 全屏、折叠与内容高度
│  ├─ 页面布局
│  │  ├─ 单栏、双栏与多区域
│  │  ├─ Grid 与 Flex 的分工
│  │  └─ 间距、对齐与内容密度
│  └─ 响应式布局
│     ├─ 不同窗口尺寸
│     ├─ 内容溢出
│     └─ 移动端基础适配
├─ 样式
│  ├─ CSS 组织
│  │  ├─ 全局样式
│  │  ├─ 组件局部样式
│  │  └─ UI 组件库样式覆盖
│  ├─ 视觉规则
│  │  ├─ 颜色、字号、字重
│  │  ├─ 间距、圆角、边框
│  │  └─ 层级、阴影、动画
│  └─ 状态样式
│     ├─ hover、focus、active
│     ├─ disabled、loading
│     └─ success、warning、error
└─ 主题
   ├─ 设计变量
   │  ├─ 颜色变量
   │  ├─ 尺寸变量
   │  └─ 组件语义变量
   ├─ 主题模式
   │  ├─ 明亮与暗黑
   │  ├─ 品牌色
   │  └─ 侧边栏与内容区风格
   └─ 主题运行机制
      ├─ CSS Variables
      ├─ 用户偏好
      └─ 初始化、切换与持久化
```

这里关注的是页面最终如何呈现。表格、表单、弹窗、树、分页等具体控件不作为本层的
一级分类，它们属于 UI 组件库提供的具体表现形式。

```text
数据交互

│  ├─ 分页、筛选与排序

│  └─ 上传、下载与导出
├─ 请求管理
│  ├─ 请求客户端封装

│  ├─ 并发、取消与去重
│  ├─ 超时与重试
```

这一方向区分“客户端自己的状态”和“服务器提供的数据”。发送请求只是起点，真正需要

### 3.4 用户与权限

```text
用户与权限
├─ 用户身份
│  ├─ 当前用户
│  ├─ 用户资料
│  ├─ 角色
│  └─ 权限集合
├─ 会话
│  ├─ 登录与退出
│  ├─ Access 凭证
│  ├─ Refresh 会话
│  ├─ 过期与续期
│  └─ 页面刷新与多标签页协调
├─ 认证
│  ├─ 未登录识别
│  ├─ 登录恢复
│  ├─ 请求身份携带
│  ├─ 认证失败处理
│  └─ 终止会话
├─ 授权
│  ├─ 角色与权限码
│  ├─ 菜单权限
│  ├─ 页面权限
│  ├─ 按钮权限
│  ├─ 接口权限
│  └─ 数据范围
└─ 安全边界
   ├─ 前端显隐不等于安全校验
   ├─ 后端是最终授权者
   ├─ Cookie、CSRF 与跨域
   ├─ Token 保存边界
   └─ XSS 与敏感信息暴露
```

这一方向需要先区分两个概念：

- 认证解决“你是谁”。
- 授权解决“你可以做什么”。

前端负责用户体验和访问引导，后端负责最终的安全校验。

## 4. 五个方向之间的关系

五个方向不是五条互不相关的路线：

```text
用户操作
  ↓
视图呈现
  ↓
应用组织
  ↓
数据交互
  ↓
后端服务

用户与权限
  └─ 横跨视图、路由、状态和请求

工程与交付
  └─ 为全部方向提供开发、测试、构建、部署和诊断保障
```

例如一次“编辑用户”操作会同时经过：

```text
视图呈现    -> 展示编辑入口和操作反馈
应用组织    -> 组件打开表单并管理页面状态
数据交互    -> 提交更新请求并刷新用户列表
用户与权限  -> 判断按钮是否可见，后端判断请求是否允许
工程与交付  -> 用类型、测试、日志和部署机制保证流程可靠
```

所以，具体业务模块适合作为贯穿五个方向的练习载体，不需要再成为第六个技术方向。

## 5. 如何使用这份地图

学习或复盘一个功能时，可以依次询问：

1. 它最终怎样呈现？
2. 它在应用中怎样组织？
3. 它怎样和服务器交换数据？
4. 哪些用户可以访问和操作？
5. 怎样验证、发布和诊断它？

能够从这五个角度完整回答，说明对该功能的前端实现已经形成闭环。
