# Pinia 与 Pinia Colada 的职责边界

## 当前项目怎么用

当前项目同时使用 Pinia 和 Pinia Colada，但它们负责的状态不同。

| 类型       | 工具               | 示例                                               |
| ---------- | ------------------ | -------------------------------------------------- |
| 本地状态   | Pinia / composable | 当前页登录态、当前用户快照、布局、主题、权限快照   |
| 服务端状态 | Pinia Colada       | 当前用户资料、用户列表、角色列表、菜单树、字典数据 |

当前已实现的服务端状态：

```text
auth.current-user
```

当前已实现的动作状态：

```text
auth.login
```

## 什么是本地状态

本地状态由前端拥有，或者至少由前端决定如何展示。

常见例子：

- 当前主题
- 菜单是否折叠
- 当前语言
- 登录态是否存在
- 页面内存中的短期 access token
- 浏览器管理且 JavaScript 不可读的 refresh Cookie
- 当前用户快照
- 页面标签页

这些状态更适合 Pinia 或普通 composable。

## 什么是服务端状态

服务端状态的权威来源在后端。

常见例子：

- 当前用户资料
- 用户列表
- 角色列表
- 权限树
- 菜单树
- 数据字典
- 报表数据

这些状态需要考虑缓存、刷新、过期、失效和多组件复用，更适合 Pinia Colada。

## 为什么不全部放进 Pinia

如果把服务端状态全部放进 Pinia，store 很快会变成这样：

```text
users
usersLoading
usersError
usersLastFetchAt
roles
rolesLoading
rolesError
menus
menusLoading
menusError
...
```

问题是：

- loading/error 状态重复
- 缓存过期要自己维护
- 新增/编辑/删除后刷新列表要自己写
- 多个页面读取同一接口时容易重复请求
- store 会同时承担本地状态和接口缓存两种职责

这会降低项目的学习清晰度。

## 为什么不全部放进 Pinia Colada

Pinia Colada 不适合管理所有状态。

这些状态不应该交给它：

- 侧边栏是否折叠
- 当前主题色
- 当前语言
- 是否打开设置面板
- 当前选中的 tab
- 表单草稿

这些不是“服务端数据”，没有必要使用 query/mutation。

## 和其它方案比较

| 方案                   | 优点                     | 缺点                           | 当前结论           |
| ---------------------- | ------------------------ | ------------------------------ | ------------------ |
| 只用 Pinia             | 容易理解，依赖少         | 服务端缓存、失效、刷新都要手写 | 不适合中型后台管理 |
| 只用 Pinia Colada      | 服务端状态清晰           | 本地 UI 状态不适合放进去       | 不采用             |
| Pinia + Pinia Colada   | 本地状态和服务端状态分开 | 需要先理解边界                 | 当前采用           |
| Pinia + TanStack Query | 能力强，生态成熟         | 对当前 Vue + Pinia 学习线稍重  | 后续可对照学习     |

## 当前项目边界规则

| 问题                               | 放哪里                               |
| ---------------------------------- | ------------------------------------ |
| 这个状态是否由后端返回？           | 是，优先 Pinia Colada                |
| 这个状态是否要缓存/刷新/失效？     | 是，优先 Pinia Colada                |
| 这个状态是否只影响 UI 展示？       | 是，优先 Pinia/composable            |
| 这个状态是否要跨页面保持？         | 是，考虑 Pinia + localStorage        |
| 这个状态是否只是某个组件内部表单？ | 是，组件内 `reactive` / `shallowRef` |

## 推荐记忆方式

```text
Pinia:
  我是谁、我当前界面是什么状态

Pinia Colada:
  后端给了我什么数据，这些数据什么时候该刷新
```
